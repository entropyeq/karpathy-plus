# Regressions

Running log of corrections and the rules they crystallized. When the user corrects the same kind of mistake twice, write the rule here so future sessions load it and stop repeating it.

One entry per rule: the rule, why it exists, and how to apply it.

## (example) Verify before claiming a file was written
**Why:** reported a save that had not actually landed.
**How to apply:** read the file back or `ls` it before stating it was saved.
